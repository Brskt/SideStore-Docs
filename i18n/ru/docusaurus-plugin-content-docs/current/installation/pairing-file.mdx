Используя JitterbugPair (*не* приложение 'Jitterbug') мы можем создать файл сопряжения для приложений таких как Sidestore для взаимодействия с вашим устройством. Это необходимо для работы SideStore.

Установите JitterbugPair для [Windows (64-bit)](https://github.com/osy/Jitterbug/releases/download/v1.3.1/jitterbugpair-win64.zip), [macOS](https://github.com/SideStore/SideStore-Docs/releases/download/need-a-place-to-put-jittterbug/jitterbugpair), или [Linux](https://github.com/osy/Jitterbug/releases/download/v1.3.1/jitterbugpair-linux.zip).

## Инструкция по сопряжению

:::tip

При использовании облачного хранилища расширение файла может измениться (обычно на .txt). Мы рекомендуем архивировать файл сопряжения перед переносом. SideStore принимает файлы только с расширением .mobiledevicepairing.
:::

### Windows

1. Распакуйте zip-архив JitterbugPair.
2. Подключите Ваше устройство к Вашему ПК и разблокируйте Ваше устройство. После этого, запустите программу.
    * Ваше устройство может запросить подтверждение о доверии Вашему ПК. Разрешите, это требуется для сопряжения.
3. После этого, Вы получите файл с расширением `.mobiledevicepairing`.
4. Перенесите этот файл через iTunes или любой другой метод. Архивируйте файл перед переносом, чтобы файл не сломался при переносе.
    * При использовании облачного хранилища расширение файла может измениться (обычно на .txt), так-что будьте аккуратны. Вы также можете изменить расширение файла на `.plist` для использования со старыми версиями SideStore, например `0.1.1`.

На Windows, Вам возможно придётся запустить программу через командную строку или Powershell, так как иногда оно может не запускаться двойным нажатием по `.exe` файлу.

5. В проводнике, перейдите в папку где находиться jitterbugpair.
6. Находясь в папке, нажмите в пустое место в панеле навигации и впишите `powershell`. Это должно открыть (возможно синего цвета) окно с названием 'PowerShell'.
7. Впишите там `./jitterbugpair.exe` и нажмите Enter, когда устройство подключено к ПК.


### macOS

1. **Распакуйте** `Jitterbugpair-macos.zip`.
2. **Поставьте пароль** на Ваше устройство если Вы этого ещё не сделали. Разблокируйте устройство и подключите его к ПК через кабель. Если Ваше устройство запросит подтверждение о доверии Вашему ПК, разрешите.
3. Откройте устройство на главном экране.
4. Найдите распакованный файл `jitterbugpair` (у него должен быть черно-зеленый значок).
5. Откройте Терминал (в Launchpad, он находится в папке "Utilities").
6. Перетащите файл 'jitterbugpair' в терминал и нажмите "Return" или Enter.
7. Если Вы получите ошибку "macOS не может подтвердить, что это приложение не содержит вредоносных программ":
   - Зайдите в Настройки > Конфиденциальность и безопасность
   - Долистайте до сообщения о программе
   - Нажмите "В любом случае открыть"
   - Попробуйте запустить программу ещё раз
8. JitterBugPair создаст **файл сопряжения** с расширением `.mobiledevicepairing`.
9. Если Вы не можете найти файл сопряжения:
   - Скопируйте название файла сопряжения
   - Вставьте его в Finder
   - Если вы запустили программу более одного раза, все файлы сопряжения должны появиться, так как они имеют одно и то же имя
10. **Перенесите файл сопряжения** на Ваше устройство iOS device через AirDrop, iCloud/One/Google Drive, почту, или любой другой метод. Мы рекомендуем архивировать файл перед переносом.

### Linux

Мы предполагаем, что вы знакомы с командной строкой Linux.

1. Распакуйте архив Jitterbug и откройте терминал (если Вы этого ещё не сделали) в распакованной папке. 
2. В этом терминале, напишите `chmod +x ./jitterbugpair`
3. Подключите Ваше устройство к Вашему ПК и разблокируйте Ваше устройство. После этого, запустите программу в Вашем терминале: `./jitterbugpair`
    * Ваше устройство может запросить подтверждение о доверии Вашему ПК, разрешите. Возможно, нужно будет перезапустить jitterbugpair.
4. После этого, Вы получите файл с расширением `.mobiledevicepairing` в папке из который Вы запустили jitterbugpair.
5. Перенесите этот файл через iTunes или любой другой метод. Архивируйте файл перед переносом, чтобы файл не сломался при переносе.
    * При использовании облачного хранилища расширение файла может измениться (обычно на .txt), так-что будьте аккуратны. Вы также можете изменить расширение файла на `.plist` для использования со старыми версиями SideStore, например `0.1.1`.

Возможно программа сработает, если добавить права на выполнение и запустив jitterbugpair из файлового менеджера, но рекомендуется делать это из терминала.
